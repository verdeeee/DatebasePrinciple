关系型数据库指采用了关系模型来组织数据的数据库。当前主流的关系型数据库有Oracle、DB2、Microsoft SQL Server、Microsoft Access、MySQL、Sybase等。

Oracle
优点：
1. 性能最高。
2. 支持多种数据类型，支持最广泛的操作系统平台。
3. 有很好的并行处理功能。
适用场景：主要满足企业开发大型数据库的需求。


DB2
优点：
1. 具有较好的可伸缩性，可支持从大型机到单用户环境，应用于所有常见的服务器操作系统平台下。
2. 采用数据分级技术。
3. 拥有完备的查询优化器。
4. 具有很好的网络支持能力。
适用场景：主要应用于大型应用系统。

Microsoft SQL Server
优点：使用方便、可伸缩性好、效率高、相关软件集成程度高（与Windows操作系统紧密集成）。
缺点：只能在Windows系统下运行。
适用场景：可作为中型企业或单位的数据库平台。


Microsoft Access
优点：
1. 界面友好、易学易用、开发简单、接口灵活、可视性好。
2. 可以与Office集成实现无缝连接。
3. 强大的数据处理功能。
4. 强大的数据组织、用户管理、安全检查等功能。
缺点：为入门级小型桌面数据库，因此性能安全性都很一般。
适用场景：可供个人管理或小型网站使用。

MySQL
优点：在WEB应用方面十分强大，速度快、灵活性强、总体拥有成本低、开放源码。
适用场景：主要用于中小型网站开发。

Sybase
优点：
1. 具有较高的数据安全性。
2. 可以与非Sybase数据源及服务器集成，允许在多个数据库之间复制数据，适于创建多层应用。
缺点：新版本具有较好的并行性，速度快，对巨量数据无明显影响，但是技术实现复杂，需要程序支持，伸缩性有限。
适用场景：在我国大中型系统中有广泛应用，适用于安全性要求极高的系统。
